## What is Normalization?

**Normalization** is the process of organizing data in a database to:
- Reduce redundancy (duplicate data)
- Eliminate anomalies (insert, update, delete problems)
- Ensure data integrity

## First Normal Form (1NF)

### Definition

A table is in **First Normal Form (1NF)** if:
1. Each column contains only **atomic** (indivisible) values
2. Each column contains values of a **single type**
3. Each row is **unique** (has a primary key)
4. There are no **repeating groups** or arrays

### What is an Atomic Value?

An **atomic value** is a value that cannot be divided further in the context of the database.

## Second Normal Form (2NF)

### Definition

A table is in **Second Normal Form (2NF)** if:
1. It is in **1NF**
2. Every non-key attribute is **fully functionally dependent** on the **entire** primary key (no partial dependencies)

> ğŸ”‘ **Key Point**: 2NF only applies to tables with a **composite primary key**. If a table has a single-column primary key, it's automatically in 2NF if it's in 1NF.

## Third Normal Form (3NF)

### Definition

A table is in **Third Normal Form (3NF)** if:
1. It is in **2NF**
2. No non-key attribute depends on another non-key attribute (no transitive dependencies)

## Boyce-Codd Normal Form (BCNF)

### Definition

A table is in **Boyce-Codd Normal Form (BCNF)** if:
1. It is in **3NF**
2. For every functional dependency X â†’ Y, X must be a **superkey** (a candidate key or a superset of a candidate key)

### When Does 3NF â‰  BCNF?

The difference appears when:
- There are multiple candidate keys
- The candidate keys overlap (share attributes)
- A non-key attribute determines part of a candidate key

### SQL for BCNF

```sql
-- Tables in BCNF

CREATE TABLE ProfessorCourse (
    Professor VARCHAR(100) PRIMARY KEY,
    Course VARCHAR(100)
);

CREATE TABLE StudentProfessor (
    StudentID VARCHAR(10),
    Professor VARCHAR(100),
    PRIMARY KEY (StudentID, Professor),
    FOREIGN KEY (Professor) REFERENCES ProfessorCourse(Professor)
);

-- Query to find student's course with a professor:
SELECT sp.StudentID, pc.Course, sp.Professor
FROM StudentProfessor sp
JOIN ProfessorCourse pc ON sp.Professor = pc.Professor;
```

### Final Schema

```sql
-- Complete normalized schema (BCNF)

CREATE TABLE Department (
    DeptID VARCHAR(10) PRIMARY KEY,
    DeptLocation VARCHAR(100)
);

CREATE TABLE Employee (
    EmpID VARCHAR(10) PRIMARY KEY,
    EmpName VARCHAR(100)
);

CREATE TABLE Project (
    ProjID VARCHAR(10) PRIMARY KEY,
    ProjName VARCHAR(100),
    DeptID VARCHAR(10),
    FOREIGN KEY (DeptID) REFERENCES Department(DeptID)
);

CREATE TABLE EmpProject (
    EmpID VARCHAR(10),
    ProjID VARCHAR(10),
    PRIMARY KEY (EmpID, ProjID),
    FOREIGN KEY (EmpID) REFERENCES Employee(EmpID),
    FOREIGN KEY (ProjID) REFERENCES Project(ProjID)
);

CREATE TABLE EmpSkill (
    EmpID VARCHAR(10),
    Skill VARCHAR(100),
    PRIMARY KEY (EmpID, Skill),
    FOREIGN KEY (EmpID) REFERENCES Employee(EmpID)
);
```
